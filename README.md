# journal-entry-analysis — 분개장 분석 플러그인

> 📊 **프로젝트 상세 소개(개발 과정·기술적 의사결정)**: [노션 포트폴리오](https://app.notion.com/p/3989016d66a981578a2dde511e02ddee)

## 프로젝트 개요
회사의 분개장(전표) 파일(CSV/Excel)을 입력하면 AI가 이상 전표를 자동 탐지하고,
계정과목별 추세·현금흐름·재무비율 분석과 권고 후속 절차를 담은 **분개장 분석 보고서**
(PDF·Word·Markdown)를 생성하는 플러그인입니다. 감사 실무와 기업 내부 점검·경영 분석에
모두 사용할 수 있으며, ERP별로 다른 분개장 헤더·컬럼명을 자동 매핑합니다.

### 탐지 항목
- **적요 이상**: 빈 적요, 무의미한 적요(test 등), 계정과목과 불일치
- **날짜 이상**: 회계기간 외·미래 일자, 입력일과 회계일의 큰 괴리(소급 입력)
- **주말·공휴일 분개**: 토·일요일 및 한국 법정 공휴일(대체공휴일 포함) 입력 전표
- **결산전표 이상**: 기말 외 결산전표, 결산 성격이 아닌 계정 사용
- **취소·역분개 패턴**: 반대 분개 쌍, 반복 취소, 작성자 불일치
- **기타 정합성**: 차대변 불일치, 전표번호 중복·결번, 자기승인, 라운드 금액

### 분석 기능
- **분석적 절차**: pandas `groupby`/`pivot_table`/`merge`로 계정과목별 월별 추세를 집계하고 계정별 개별 차트(급증 월 강조)와 AI 해석문(급변 원인 전표 드릴다운·사유 추론, 이상 전표 월별 병기) 생성
- **현금흐름 분석**: 영업·투자·재무 활동별 월별 현금흐름 표·차트, 차입금 변동(차입·상환·대환 가능성) 상세 내역과 해석
- **재무비율 계산**: 유동비율, 재고자산회전율, 매출채권회전율 등을 계정 잔액 집계로 산출하고 해석 코멘트 제공
- **리스 회계처리 검토**: 지급수수료 등 판관비로 처리된 정기 고정 지급에서 리스(K-IFRS 1116) 누락 의심 탐지
- **결과 출력**: 분개장 분석 보고서(PDF·Word·Markdown) + 원본 엑셀에 `이상전표` 시트 추가(이상유형·이상징후·문제설명 컬럼, 추가 전 자동 백업)

## 동작 방식

이 플러그인은 **2층 구조**로 동작합니다.

| 층 | 담당 | 이유 |
|---|---|---|
| **결정적 파이프라인** (`scripts/analyze_journal.py`) | 컬럼 매핑, 이상 탐지 A~L, 추세 집계, 현금흐름, 재무비율, 차트·보고서·시트 생성 | 누가 몇 번을 돌려도 같은 입력 → 같은 탐지 결과 보장 (AI가 즉석에서 코드를 재작성하지 않음) |
| **AI 판단 계층** (SKILL.md 지시) | 컬럼 매핑 검증·사용자 확인, **부정 위험 정황 스캔**(규칙 밖 의심 정황 감지), 자동 해석문 검토·보강, 권고 절차 구체화 | 적요 검토 과정에서 회계처리 의심 정황이 포착되는 경우의 판단 담당. 예: 계정과목은 '지급수수료'로 전액 당기 비용 처리됐지만 적요가 "연간 소프트웨어 라이선스 이용료"인 전표 — 이용 기간이 남았다면 선급비용으로 자산화한 뒤 기간에 걸쳐 나눠 비용화해야 하는 회계처리 이슈 |

지식은 3단계로 분리 배치되어 필요할 때만 로드됩니다(progressive disclosure):
`SKILL.md`(트리거·워크플로) → `references/`(점검 정의·매핑 규칙) → `scripts/`(실행 코드).

## 처리 절차

1. **입력 파악·전처리** — 헤더 행 자동 탐지(상위 8행 중 헤더 키워드 최다 행 선택), ERP별 컬럼명을 동의어 사전으로 표준 필드에 매핑, 날짜를 내용 검증 기반으로 datetime 변환, 주말·공휴일(대체공휴일 포함) 파생
2. **이상 전표 탐지 (12종, A~L)** — 적요 이상 / 기간외·날짜오류 / 주말·공휴일 / 결산전표 이상 / 취소·역분개 / 차대불일치·중복·자기승인·라운드 거액 / 적요-계정 불일치 / 근접일 중복 / 가지급금 미정산 / 거액 증빙 누락 / 비경상 대체(부채→잡이익) / 리스 회계처리 누락
3. **AI 부정 위험 정황 스캔 (M)** — 5개 관점의 의심 정황 감지: 위법·부정 암시 적요(리베이트·사례비·커미션), 특수관계자 정황(임원·가족 개인성 지출), 권한·통제 우회(평소와 다른 부서의 기표·승인), 금액 쪼개기(승인 한도 회피 분할), 거래처 패턴(저빈도 거래처 거액). **대량 데이터 대응 설계**: 후보 추출은 스크립트가 결정적으로 수행해 `AI정황_후보.md`(관점별 금액 상위 50건 상한)로 저장하고, AI는 원시 데이터가 아닌 **후보 목록만 판독** — 토큰 소비가 거래 건수(100만 건이든)와 무관하게 후보 수에 비례. 발견 시 이상전표 시트에 `AI정황` 유형으로 추가, 상한 초과 생략분은 건수 명시
4. **분석적 절차** — 계정과목별 월별 추세 차트(급증 월 강조) + 급변 원인 전표 드릴다운 → 적요 기반 사유 추론 → 이상 전표 월·유형 병기
5. **현금흐름·재무비율** — 영업·투자·재무 활동별 월별 흐름, 차입금 변동(차입/상환, 7일 내 근접 시 대환 가능성 표시), 유동비율·매출채권회전율·회수기간·재고자산회전율(계산 가능 여부와 사유 명시)
6. **산출물 생성 + AI 검토** — 원본 엑셀에 `이상전표` 시트(백업 후), 분석 보고서 PDF·Word·Markdown 생성. AI가 보고서를 읽고 해석·권고를 실제 사례(전표번호·금액 인용)로 보강한 뒤 사용자에게 위험도 순으로 보고

## 내장 지식

- **ERP 매핑 지식**: 표준 필드별 동의어 사전(회계일 ← 전기일/기표일자/Posting Date 등), 금액 단일컬럼+차대구분 구조 분리, 날짜 3형식(문자열·YYYYMMDD 정수·Excel 직렬값) 판별
- **한국 세무·회계 일정**: 법정 공휴일(대체공휴일 포함), 분기 부가세 신고월(1·4·7·10월) — 신고 정리로 인한 급증을 이상으로 오판하지 않음
- **회계 처리 관행**: 감가상각 월할 계상 정책 인식(6개월 이상 규칙 발생 시 정상 간주), 결산 성격 계정 목록, 계정과목명 기반 유동자산/유동부채/활동(영업·투자·재무) 분류
- **회계기준**: K-IFRS 1116 리스(단기·소액 제외 사용권자산·리스부채 계상) vs K-GAAP 운용리스 허용 — 적용 기준에 따라 지적 수위 분기
- **부정 위험 패턴**: 기말일 대형 취소(이익조정), 기간귀속 조작(cut-off), 자기승인, 가지급금 미정산(인정이자), 가공경비(증빙 누락 거액) 등 규칙화된 패턴 + **AI 정황 스캔**으로 규칙 밖 의심 정황(리베이트성 적요, 특수관계자 지출, 권한 우회, 금액 쪼개기, 거래처 이상 패턴)까지 감지

## 판단 기준 (주요 문턱값)

| 점검 | 기준 |
|---|---|
| 적요 이상 | 공란, 무의미 문자열, 3자 미만 |
| 라운드 거액 | 5천만원 이상 & 1천만원 단위 (기말 7일 이내면 강조) |
| 근접일 중복 | 동일 계정·금액·적요 차변(1백만↑)이 7일 이내 반복 |
| 거액 증빙 누락 | 증빙 공란 & 5백만원 이상 (접대비 등 민감 계정 강조) |
| 리스 의심 | 리스·렌탈 적요 키워드 또는 유사 금액(변동계수 ≤0.10) 6개월 이상 매월 반복 |
| 월할 정책 인정 | 6개월 이상 매월 발생 → 정상, 이탈(말일 아님·같은 달 2회)만 플래그 |
| 추세 급변 | 전월 대비 ±35% & 변동 2천만↑ 또는 월 중앙값 1.7배 & 3천만↑ (이중 기준) |

**공통 원칙**
1. 결과는 "판정"이 아닌 **"탐지"** — 최종 판단은 담당자 소명·증빙 확인 후 이루어져야 함을 산출물에 명시
2. **오탐 억제** — 규칙적 정상 패턴(월할 상각, 분기 부가세, 매월 급여)은 정책·주기로 인식해 제외하고 이탈분만 플래그
3. 수치는 반드시 스크립트 계산 결과만 사용 (임의 추정 금지)
4. AI 정황 스캔 결과는 **정황이지 확증이 아님** — 데이터 근거(전표번호·금액·날짜)와 함께 "확인 필요" 형태로 서술

## 정보가 부족하거나 실패하는 상황에서의 동작

| 상황 | 동작 |
|---|---|
| 필수 컬럼 매핑 실패 | 스크립트가 컬럼 목록과 함께 **안전하게 중단** → AI가 데이터 샘플로 매핑을 추정해 **사용자에게 확인받은 뒤** 재실행 |
| 날짜 변환 성공률 80% 미만 | 전체 컬럼을 내용 기반 재탐색(연도 타당성 1990~2100 검증), 그래도 실패 시 중단 후 사용자 확인 |
| 선택 컬럼(증빙·기표자·승인자) 없음 | 해당 점검(증빙 누락·자기승인)만 생략하고 나머지 계속 진행 |
| CSV 인코딩 문제 | utf-8-sig → cp949 → euc-kr 순 자동 재시도 |
| holidays / python-docx 미설치 | 공휴일 점검·Word 생성만 생략하고 경고 출력 (공휴일은 목록 직접 정의로 대체 가능) |
| Edge 브라우저 없음 | PDF 생략, HTML을 브라우저에서 수동 변환하도록 안내 |
| 입력이 CSV(시트 추가 불가) | 별도 `이상전표_<날짜>.xlsx` 생성 |
| 기초잔액 없음(분개장 특성) | 잔액이 아닌 "당기 순증감" 기준임을 보고서에 명시 |
| 매출원가 등 관련 계정 부재 | 해당 재무비율 "계산 불가"로 표기하고 사유 기재 (임의 추정 금지) |
| 판단이 갈리는 사례 (예: 상각누계액 차변 = 이중계상 vs 자산 처분) | 단정하지 않고 **대안 해석 병기** |
| 회계연도 오추정 | `--fy` 인자로 재실행 |
| **어떤 실패에서도** | 원본 시트 데이터는 불변 — 허용되는 변경은 `이상전표` 시트 추가뿐이며 추가 전 자동 백업, 기존 백업이 있으면 덮어쓰지 않음 |

## 폴더 구조
```

├── src/
│   ├── .codex-plugin/plugin.json           # 플러그인 매니페스트
│   ├── skills/journal-entry-analysis/
│   │   ├── SKILL.md                        # 스킬 정의 (워크플로·AI 개입 지점)

│   │   └── references/
│   │       ├── checks.md                   # 점검 항목 상세·문턱값·오탐 억제 규칙

│   └── .mcp.json                           # MCP 서버 설정

├── logs/                                   # 세션 대화 로그 (<tool>/<session_id>.jsonl)
└── README.md
```

## 사용 방법
1. 플러그인(또는 스킬)을 설치한 뒤 분개장 파일을 준비합니다. (예시: `sample_data/예시분개장1.xlsx`)
2. 프롬프트에서 요청합니다. 예:
   - "sample_data/예시분개장1.xlsx 분개장에서 이상 전표를 찾아줘"
   - `$journal-entry-analysis 2025년 전표 검토해줘`
3. AI가 컬럼을 매핑하고 항목별 점검·분석을 수행한 뒤 원본에 `이상전표` 시트를 추가하고
   `분석보고서_<날짜>.pdf/docx/md`와 차트를 생성합니다.
4. 탐지 성능: 동봉된 `sample_data/정답지.xlsx` 기준 27/27 탐지 (검증 완료)

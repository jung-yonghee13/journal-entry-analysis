# journal-entry-analysis — 분개장 분석 플러그인

![License: MIT](https://img.shields.io/badge/License-MIT-green.svg) ![Python](https://img.shields.io/badge/Python-3.10%2B-blue.svg)

> 📊 **프로젝트 상세 소개(개발 과정·기술적 의사결정)**: [노션 포트폴리오](https://app.notion.com/p/3989016d66a981578a2dde511e02ddee)

## 프로젝트 개요
회사의 분개장(전표) 파일을 넣으면, AI가 이상 전표를 자동으로 찾아내고 계정과목별 추세·현금흐름·재무비율 분석과 후속 조치 권고까지 담은 **분개장 분석 보고서**(PDF·Word·Markdown)를 만들어 드립니다. 감사 실무는 물론 기업 내부 점검이나 경영 분석에도 두루 쓸 수 있도록 설계했고, ERP마다 제각각인 분개장 헤더와 컬럼명은 자동으로 맞춰 줍니다. CSV와 Excel을 모두 지원합니다.

### 탐지 항목
- **적요 이상**: 빈 적요, 무의미한 적요(test 등), 계정과목과 불일치
- **날짜 이상**: 회계기간 외·미래 일자, 입력일과 회계일의 큰 괴리(소급 입력)
- **주말·공휴일 분개**: 토·일요일 및 한국 법정 공휴일(대체공휴일 포함) 입력 전표
- **결산전표 이상**: 기말 외 결산전표, 결산 성격이 아닌 계정 사용
- **취소·역분개 패턴**: 반대 분개 쌍, 반복 취소, 작성자 불일치
- **기타 정합성**: 차대변 불일치, 전표번호 중복·결번, 자기승인, 라운드 금액

### 분석 기능
- **분석적 절차**: pandas `groupby`/`pivot_table`/`merge`로 계정과목별 월별 추세를 집계하고, 계정별 개별 차트(급증 월 강조)와 AI 해석문(급변 원인 전표 드릴다운·사유 추론, 이상 전표 월별 병기)을 만들어 드립니다.
- **현금흐름 분석**: 영업·투자·재무 활동별 월별 현금흐름 표·차트와 차입금 변동(차입·상환·대환 가능성) 내역을 해석과 함께 정리합니다.
- **재무비율 계산**: 유동비율, 재고자산회전율, 매출채권회전율 등을 계정 잔액 집계로 산출하고 해석 코멘트를 덧붙입니다.
- **리스 회계처리 검토**: 지급수수료 등 판관비로 처리된 정기 고정 지급에서 리스(K-IFRS 1116) 누락 가능성을 짚어 드립니다.
- **결과 출력**: 분개장 분석 보고서(PDF·Word·Markdown)와 함께, 원본 엑셀에 `이상전표` 시트(이상유형·이상징후·문제설명 컬럼)를 더해 드립니다. 시트를 추가하기 전에 원본은 자동으로 백업합니다.

## 동작 방식

이 플러그인은 성격이 다른 두 층으로 나뉘어 움직입니다. 계산과 탐지는 규칙을 그대로 따르는 파이프라인이 맡고, 규칙으로는 잡기 어려운 판단은 AI가 맡습니다.

| 층 | 담당 | 이렇게 나눈 이유 |
|---|---|---|
| **규칙 기반 파이프라인** (`scripts/analyze_journal.py`) | 컬럼 매핑, 이상 탐지 A~L, 추세 집계, 현금흐름, 재무비율, 차트·보고서·시트 생성 | 같은 파일을 넣으면 누가 몇 번을 돌려도 같은 결과가 나오도록, AI가 그때그때 코드를 다시 짜지 않게 했습니다. 분석 결과의 재현성을 위해서입니다. |
| **AI 판단 계층** (SKILL.md 지시) | 컬럼 매핑 검증·사용자 확인, **부정 위험 정황 스캔**(규칙 밖 의심 정황 감지), 자동 해석문 검토·보강, 권고 절차 구체화 | 적요를 읽어야 비로소 보이는 이슈를 맡습니다. 예를 들어 계정과목은 '지급수수료'로 전액 당기 비용 처리됐지만 적요는 "연간 소프트웨어 라이선스 이용료"인 전표라면, 이용 기간이 남은 만큼 선급비용으로 자산화해 기간에 걸쳐 나눠 비용화해야 하는지 검토가 필요합니다. |

지침과 지식은 세 단계로 나눠 두어 필요할 때만 불러오도록 했습니다(progressive disclosure):
`SKILL.md`(트리거·워크플로) → `references/`(점검 정의·매핑 규칙) → `scripts/`(실행 코드).

## 처리 절차

1. **입력 파악·전처리** — 상위 8행에서 헤더 키워드가 가장 많은 줄을 헤더로 잡고, ERP별 컬럼명을 동의어 사전으로 표준 필드에 매핑합니다. 날짜는 값의 내용까지 검증해 datetime으로 변환하고, 주말·공휴일(대체공휴일 포함) 여부를 파생합니다.
2. **이상 전표 탐지 (12종, A~L)** — 적요 이상 / 기간외·날짜오류 / 주말·공휴일 / 결산전표 이상 / 취소·역분개 / 차대불일치·중복·자기승인·라운드 거액 / 적요-계정 불일치 / 근접일 중복 / 가지급금 미정산 / 거액 증빙 누락 / 비경상 대체(부채→잡이익) / 리스 회계처리 누락을 점검합니다.
3. **AI 부정 위험 정황 스캔 (M)** — 다섯 관점의 의심 정황을 살핍니다: 위법·부정 암시 적요(리베이트·사례비·커미션), 특수관계자 정황(임원·가족 개인성 지출), 권한·통제 우회(평소와 다른 부서의 기표·승인), 금액 쪼개기(승인 한도 회피 분할), 거래처 패턴(저빈도 거래처 거액). **대량 데이터를 위한 설계**로, 후보 추출은 스크립트가 결정적으로 수행해 `AI정황_후보.md`(관점별 금액 상위 50건 상한)로 저장하고, AI는 원시 데이터가 아니라 **이 후보 목록만** 읽습니다. 덕분에 토큰 소비가 거래 건수(100만 건이라도)와 무관하게 후보 수에만 비례합니다. 걸린 전표는 이상전표 시트에 `AI정황` 유형으로 더하고, 상한을 넘겨 생략된 건은 건수를 명시합니다.
4. **분석적 절차** — 계정과목별 월별 추세 차트(급증 월 강조)를 그리고, 급변한 달의 원인 전표를 드릴다운해 적요를 근거로 사유를 추론하며, 이상 전표는 월·유형과 함께 병기합니다.
5. **현금흐름·재무비율** — 영업·투자·재무 활동별 월별 흐름과 차입금 변동(차입/상환, 7일 내 근접 시 대환 가능성 표시)을 정리하고, 유동비율·매출채권회전율·회수기간·재고자산회전율을 계산 가능 여부·사유와 함께 제시합니다.
6. **산출물 생성 + AI 검토** — 원본 엑셀에 `이상전표` 시트를 더하고(백업 후) 분석 보고서를 PDF·Word·Markdown으로 만듭니다. 이어서 AI가 보고서를 읽고 해석과 권고를 실제 사례(전표번호·금액 인용)로 보강한 뒤, 위험도 순으로 정리해 전달합니다.

## 내장 지식

- **ERP 매핑 지식**: 표준 필드별 동의어 사전(회계일 ← 전기일/기표일자/Posting Date 등), 금액 단일컬럼+차대구분 구조 분리, 날짜 3형식(문자열·YYYYMMDD 정수·Excel 직렬값) 판별을 담았습니다.
- **한국 세무·회계 일정**: 법정 공휴일(대체공휴일 포함)과 분기 부가세 신고월(1·4·7·10월)을 알고 있어, 신고 정리로 인한 급증을 이상으로 오판하지 않습니다.
- **회계 처리 관행**: 감가상각 월할 계상을 정상 방침으로 인식(6개월 이상 규칙 발생 시)하고, 결산 성격 계정 목록과 계정과목명 기반의 유동자산/유동부채/활동(영업·투자·재무) 분류를 갖추고 있습니다.
- **회계기준**: K-IFRS 1116 리스(단기·소액 제외 사용권자산·리스부채 계상)와 K-GAAP 운용리스 허용을 구분해, 적용 기준에 따라 지적 수위를 달리합니다.
- **부정 위험 패턴**: 기말일 대형 취소(이익조정), 기간귀속 조작(cut-off), 자기승인, 가지급금 미정산(인정이자), 가공경비(증빙 누락 거액) 같은 규칙화된 패턴을 잡고, 여기에 **AI 정황 스캔**을 더해 규칙 밖 의심 정황(리베이트성 적요, 특수관계자 지출, 권한 우회, 금액 쪼개기, 거래처 이상 패턴)까지 살핍니다.

## 룰 기반 탐지 기준 (주요 문턱값)

아래 항목은 사람이 정한 조건·문턱값을 스크립트가 그대로 적용하는 **룰 기반(rule-based) 탐지**입니다. 같은 입력이면 언제나 같은 결과가 나오고, 규칙에 걸린 전표를 "판정"하는 것이 아니라 "플래그"만 합니다.

| 점검 | 기준 |
|---|---|
| 적요 이상 | 공란, 무의미 문자열, 3자 미만 |
| 라운드 거액 | 5천만원 이상 & 1천만원 단위 (기말 7일 이내면 강조) |
| 근접일 중복 | 동일 계정·금액·적요 차변(1백만↑)이 7일 이내 반복 |
| 거액 증빙 누락 | 증빙 공란 & 5백만원 이상 (접대비 등 민감 계정 강조) |
| 리스 의심 | 리스·렌탈 적요 키워드 또는 유사 금액(변동계수 ≤0.10) 6개월 이상 매월 반복 |
| 월할 정책 인정 | 6개월 이상 매월 발생 → 정상, 이탈(말일 아님·같은 달 2회)만 플래그 |
| 추세 급변 | 전월 대비 ±35% & 변동 2천만↑ 또는 월 중앙값 1.7배 & 3천만↑ (이중 기준) |

**공통 원칙 (룰 기반 탐지)**
1. 결과는 "판정"이 아니라 **"탐지"**입니다. 최종 판단은 담당자의 소명과 증빙 확인을 거쳐야 한다는 점을 산출물에 명시합니다.
2. **오탐 억제** — 규칙적으로 반복되는 정상 패턴(월할 상각, 분기 부가세, 매월 급여)은 정책·주기로 인식해 제외하고, 거기서 벗어난 것만 플래그합니다.
3. 보고서에 들어가는 수치는 반드시 스크립트가 계산한 값만 사용합니다(임의 추정 금지).

**AI 정황 스캔 원칙 (룰 밖 의심 정황 판독 시)**
- 룰로 정의하기 어려운 표현·문맥상의 이상은 AI가 후보를 읽고 판단하되, 그 결과는 **정황이지 확증이 아닙니다**. 반드시 데이터 근거(전표번호·금액·날짜)와 함께 "확인 필요" 형태로 서술합니다.

## 정보가 부족하거나 실패하는 상황에서의 동작

| 상황 | 동작 |
|---|---|
| 필수 컬럼 매핑 실패 | 스크립트가 컬럼 목록과 함께 **안전하게 중단** → AI가 데이터 샘플로 매핑을 추정해 **사용자에게 확인받은 뒤** 재실행합니다. |
| 날짜 변환 성공률 80% 미만 | 전체 컬럼을 내용 기반으로 재탐색(연도 타당성 1990~2100 검증)하고, 그래도 실패하면 중단 후 사용자에게 확인합니다. |
| 선택 컬럼(증빙·기표자·승인자) 없음 | 해당 점검(증빙 누락·자기승인)만 건너뛰고 나머지는 그대로 진행합니다. |
| CSV 인코딩 문제 | utf-8-sig → cp949 → euc-kr 순으로 자동 재시도합니다. |
| holidays / python-docx 미설치 | 공휴일 점검·Word 생성만 생략하고 경고를 남깁니다(공휴일은 목록 직접 정의로 대체 가능). |
| Edge 브라우저 없음 | PDF는 생략하고, HTML을 브라우저에서 직접 인쇄하도록 안내합니다. |
| 입력이 CSV(시트 추가 불가) | 별도로 `이상전표_<날짜>.xlsx`를 만듭니다. |
| 기초잔액 없음(분개장 특성) | 잔액이 아니라 "당기 순증감" 기준임을 보고서에 명시합니다. |
| 매출원가 등 관련 계정 부재 | 해당 재무비율을 "계산 불가"로 표기하고 사유를 적습니다(임의 추정 금지). |
| 판단이 갈리는 사례 (예: 상각누계액 차변 = 이중계상 vs 자산 처분) | 단정하지 않고 **두 해석을 나란히** 적습니다. |
| 회계연도 오추정 | `--fy` 인자로 다시 실행합니다. |
| **어떤 실패에서도** | 원본 시트 데이터는 그대로 둡니다. 허용되는 변경은 `이상전표` 시트 추가뿐이며, 추가 전 자동 백업하고 기존 백업이 있으면 덮어쓰지 않습니다. |

또는 Codex 앱/CLI의 `/plugins`에서 로컬 마켓플레이스를 통해 설치합니다. 플러그인 루트는 `src/`입니다.

## 사용 방법

### 1. 준비
플러그인(또는 스킬)을 설치하고 분석할 분개장 파일을 준비합니다. 동봉한 `sample_data/예시분개장1.xlsx`(2025 회계연도 가상 분개장)로 바로 시험해 볼 수 있습니다.

### 2. 실행
프롬프트에 파일 경로와 함께 요청하면 됩니다. 자연어로 부탁해도 되고, 스킬을 직접 지정해도 됩니다.
- "`sample_data/예시분개장1.xlsx` 분개장에서 이상 전표를 찾아줘"
- `$journal-entry-analysis 2025년 전표 검토해줘`

스크립트를 직접 돌릴 수도 있습니다(스킬은 내부적으로 이 명령을 실행합니다).
```
python src/skills/journal-entry-analysis/scripts/analyze_journal.py \
       --input <분개장.xlsx|csv> [--outdir <출력폴더>] [--sheet <시트명>] [--fy <회계연도>]
```
- `--sheet`, `--fy`는 생략하면 자동으로 판단합니다(분개장 시트·최빈 회계연도).

### 3. 진행 과정
실행하면 ① 헤더·컬럼 매핑과 날짜 변환 → ② 12종 룰 탐지(A~L) → ③ AI 부정 위험 정황 스캔(M) → ④ 계정별 추세·현금흐름·재무비율 분석 → ⑤ 시트 추가와 보고서 생성 순으로 진행합니다. 컬럼 매핑이 애매하거나 회계연도가 이상하면 중간에 멈추고 확인을 요청하므로, 잘못된 결과가 조용히 나오는 일은 없습니다.

> 탐지 성능은 동봉한 `sample_data/정답지.xlsx`(심어 둔 이상 전표 27건) 기준으로 **27/27**을 확인했습니다.

## 산출물

실행이 끝나면 결과가 세 갈래로 나옵니다.

### ① 원본 엑셀에 `이상전표` 시트 추가
탐지된 전표를 원본 파일 안의 새 시트로 모아, 원본 전표와 이상 목록을 한 파일에서 대조할 수 있게 합니다.
- 원본 전표의 모든 컬럼을 그대로 두고, 오른쪽에 세 컬럼을 덧붙입니다 — **이상유형**(분류), **이상징후**(한눈에 보는 요약), **문제설명**(자세한 근거). 한 전표에 문제가 여럿이면 한 줄에 함께 적습니다.
- 시트를 더하기 전에 원본은 `<파일명>_backup_<날짜>.xlsx`로 자동 백업하며, **기존 분개장 시트는 셀 하나도 바꾸지 않습니다.**
- 입력이 CSV라 시트를 더할 수 없으면 별도로 `이상전표_<날짜>.xlsx`를 만듭니다.

### ② 분석 보고서 (PDF · Word · Markdown)
`<입력파일명>_분석/` 폴더에 `분석보고서_<날짜>.pdf` / `.docx` / `.md` 세 형식으로 저장합니다. 구성은 다음과 같습니다.
- **요약**: 전체 전표 수, 이상 전표 수, 이상유형별 건수
- **주요 이상 전표**: 금액 상위 순으로 전표번호·계정과목·금액·이상징후
- **계정과목별 월별 추세와 해석**: 계정별 차트와, 급변 원인 전표를 근거로 한 사유 설명
- **현금흐름·재무비율**: 활동별 현금흐름, 차입금 변동, 유동비율·회전율 등
- **권고 절차**: 이상 유형별로 어떤 후속 조치를 하면 좋을지 실제 사례를 인용해 제시

### ③ 차트와 근거 파일
같은 `<입력파일명>_분석/` 폴더 아래에 함께 만들어집니다.
- `charts/` — 계정과목별 월별 추세 차트(급증 월 강조)와 활동별 현금흐름 차트(PNG)
- `AI정황_후보.md` — AI 정황 스캔이 검토한 후보 목록(관점별 상위 50건). AI가 어떤 근거로 정황을 판단했는지 되짚어 볼 수 있습니다.

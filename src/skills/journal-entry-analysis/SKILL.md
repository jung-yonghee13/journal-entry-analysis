# journal-entry-analysis — 분개장 분석

분개장(전표) 파일에서 이상 전표를 탐지하고 추세·현금흐름·재무비율을 분석해 **분개장 분석 보고서**를 만든다.
(기업 내부 점검용으로도 쓰이므로 "감사 보고서"라는 명칭은 쓰지 않는다.)

결정적 작업(탐지·집계·차트·보고서 생성)은 모두 `scripts/analyze_journal.py`가 수행한다.
**같은 로직을 즉석에서 재작성하지 말고 반드시 이 스크립트를 실행한다** — 실행마다 동일한
탐지 결과를 보장하기 위함이다. AI의 역할은 실행 전후의 판단·검토·보완이다.

## 실행 절차

1. **실행** — 스크립트는 **이 SKILL.md와 같은 폴더의 `scripts/analyze_journal.py`**다.
   설치된 플러그인은 작업 디렉토리 밖에 있으므로 반드시 절대경로로 실행한다:
   ```
   python "<이 스킬 폴더>/scripts/analyze_journal.py" --input <분개장.xlsx|csv> [--outdir <폴더>] [--sheet <시트명>] [--fy <연도>]
   ```
   - 헤더 행·컬럼 매핑·날짜 컬럼은 자동 탐지된다(규칙: `references/column-mapping.md`).
   - 필요 패키지: pandas, matplotlib, openpyxl, holidays, python-docx (없으면 설치 시도;
     holidays·python-docx는 없어도 해당 기능만 생략하고 진행된다)

2. **스크립트 출력 확인** — 실행 로그에서 다음을 검증한다:
   - `[매핑]` 줄: 컬럼 매핑이 타당한지. **틀렸거나 `[중단]`이 뜨면** 컬럼 목록·데이터 샘플을 보고
     매핑을 추정해 사용자에게 확인받은 뒤 재실행한다(방법: `references/column-mapping.md`의 "실패 시 처리").
   - `[기간]`·`[탐지]` 줄: 회계연도(FY)와 탐지 건수가 합리적인지. FY가 틀리면 `--fy`로 재실행.

3. **AI 부정 위험 정황 스캔** (스크립트 규칙이 못 잡는 의심 정황 — 정의: `references/checks.md`의 "M. AI 정황 스캔")
   **원시 분개장을 직접 읽지 않는다** — 100만 행급 데이터에서 토큰이 행 수에 비례해 폭발하기 때문이다.
   대신 스크립트가 출력한 `<출력폴더>/AI정황_후보.md`(관점별 후보, 금액 상위 50건 상한)만 판독한다:
   - 후보 파일의 M1~M5 관점: 부정 암시 적요 / 특수관계자 / 권한·통제 우회 / 한도 근접 금액 반복 / 저빈도 거래처 거액
   - **기탐지=N**(규칙 A~L에 안 걸린) 후보를 우선 판단하고, 기탐지=Y는 정황 해석을 보강한다.
   - M4(한도 근접 반복)는 일자 간격으로 판단한다: 근접일 연속 분할이면 쪼개기 의심, 수개월 간격이면 정상.
   - 후보 상한을 초과해 생략된 건수가 있으면 보고서에 그 사실을 명시한다(조용한 누락 금지).
   - 후보 밖에서 추가 확인이 필요하면 원시 데이터를 읽지 말고 **pandas 질의를 작성해 집계·필터 결과만** 본다.
   - 발견 시 이상전표 시트에 행을 추가하고(이상유형 `AI정황`, 이상징후 "부정 위험 정황"), 보고서에 별도
     섹션으로 서술한다. **확증이 아닌 정황임을 명시**하고, 확신이 낮으면 단정 대신 확인 질문 형태로 쓴다.

4. **보고서 검토·보강** — 생성된 `분석보고서_<날짜>.md`를 읽고:
   - 계정별 해석문과 현금흐름 해석이 데이터와 맞는지 검증하고, 어색한 자동 해석은 고쳐 쓴다.
   - 권고 절차를 이 회사 데이터의 실제 사례(금액·전표번호 인용)로 구체화한다.
   - 수정했으면 docx/html도 같은 내용으로 갱신하고 PDF를 재변환한다.

5. **사용자 보고** — 핵심 발견사항(위험도 순), 산출물 경로, 원본 보존 여부(백업 파일)를 요약해 전달한다.

## 산출물

| 산출물 | 위치 |
|---|---|
| 원본 엑셀에 `이상전표` 시트 (이상유형·이상징후·문제설명 컬럼) | 입력 파일 (추가 전 `*_backup_*.xlsx` 자동 백업) |
| 분석 보고서 (md / docx / html→pdf) | `<출력폴더>/분석보고서_<날짜>.*` |
| 계정별 추세·현금흐름 차트 | `<출력폴더>/charts/*.png` |

## 주의사항

- 원본 파일의 기존 시트 데이터는 절대 수정·삭제하지 않는다. 허용되는 변경은 `이상전표` 시트 추가뿐.
  백업이 이미 있으면 다시 만들지 않는다(시트 추가된 파일로 백업이 오염되는 것 방지).
- 이상 전표 "판정"이 아니라 "탐지"임을 항상 명시한다. 수치는 스크립트 계산 결과만 사용한다.
- 문턱값·규칙을 조정할 때는 `scripts/analyze_journal.py`와 `references/checks.md`를 함께 수정한다.

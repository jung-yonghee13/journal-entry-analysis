# ERP별 분개장 컬럼 매핑 규칙

`scripts/analyze_journal.py`의 `SYNONYMS`·`detect_header_row`·`parse_dates`가 구현하는 규칙.
새 ERP 형식을 만나면 이 문서와 스크립트의 `SYNONYMS`를 함께 갱신한다.

## 헤더 행 자동 탐지
ERP에 따라 첫 행이 제목("분개장출력" 등)이고 실제 헤더가 2~3행에 있다.
상위 8행을 `header=None`으로 읽어 **헤더 키워드가 가장 많이 등장하는 행**을 헤더로 선택한다.

## 표준 필드 동의어 사전
| 표준 필드 | 동의어 (앞이 우선) | 필수 |
|---|---|---|
| 회계일 | 회계일자, 회계일, 전표일자, 기표일자, 전기일, 증빙일, 발생일, 거래일자, 일자, Posting Date, GL Date, Date | ✔ |
| 입력일시 | 입력일시, 입력일, 등록일시, 생성일시, Entry Date, Created | |
| 회계전표번호 | 회계전표번호, 전표번호, 전표NO, 문서번호, Voucher No, Doc No | ✔* |
| 전표행번호 | 전표행번호, 행번호, 라인번호, Line No | |
| 계정과목 | 계정과목명, 계정과목, 계정명, G/L Account, Account | ✔ |
| 차변금액 / 대변금액 | 차변금액·차변액·차변·Debit·DR / 대변금액·대변액·대변·Credit·CR | ✔* |
| 금액 + 차대구분 | 전표금액·금액·Amount + 차대구분·차대·D/C | (차/대변 대체) |
| 적요 | 적요내용, 마스터적요, 적요, Description, Text | ✔ |
| 증빙 | 증빙유형, 증빙구분, 증빙 | |
| 기표자 / 승인자 | 기표자·작성자·입력자 / 승인자·결재자·Approver | |
| 기표부서 / 승인부서 | 기표부서·작성부서·입력부서 / 승인부서·결재부서 (M3 권한 우회 점검용) | |

매칭 순서: ① 정규화(공백·괄호 제거, 소문자) 완전 일치 → ② 동의어가 컬럼명에 포함.
하나의 원본 컬럼은 하나의 표준 필드에만 배정된다.

\* 대체 규칙:
- **전표번호가 없고 행번호만 있으면**: 행번호에서 끝의 `-NNN`을 떼어 전표번호 생성.
  행번호도 없으면 전표번호 + 그룹 내 순번으로 행번호 합성.
- **금액 단일 컬럼 + 차대구분 컬럼** 구조: 구분값 첫 글자(차/D/1)로 차변·대변 두 컬럼으로 분리.

## 날짜 내용 기반 검증 (`parse_dates`)
컬럼명이 매칭돼도 내용 검증을 거친다. 세 가지 형식을 모두 시도해 성공률 최고를 채택:
1. 일반 문자열/datetime → `pd.to_datetime(errors="coerce")`
2. `20250102` 형식 정수 → `format="%Y%m%d"`
3. Excel 직렬값 → `unit="D", origin="1899-12-30"` (연도 1990~2100 검증)

매핑된 컬럼의 변환 성공률이 80% 미만이면 **모든 컬럼**에 시도해 성공률 최고·연도 분포가
합리적인 컬럼을 날짜 컬럼으로 재선정한다.

## 실패 시 처리 (AI 개입 지점)
- 필수 필드 매핑 실패 → 스크립트가 컬럼 목록과 함께 중단 메시지 출력.
  **AI가 컬럼 목록과 데이터 샘플을 보고 매핑을 추정해 사용자에게 확인받은 뒤**,
  원본 컬럼명을 표준명으로 rename한 사본을 만들어 재실행하거나 `SYNONYMS`에 추가한다.
- 시트 자동 선택이 틀리면 `--sheet <시트명>` 지정.
- 인코딩: CSV는 utf-8-sig → cp949 → euc-kr 순 자동 재시도.
